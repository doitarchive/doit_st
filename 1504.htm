21.4 ExtenDOS


ExtenDOS - Treiber für CD-, CDR und CDRW Laufwerke



Anodyne-Software in Kanada liefert seit einiger Zeit CD Writer Suite aus. CD Writer Suite umfasst mehrere Programme für den bekannten CD-Treiber ExtenDOS, welcher momentan in der Version 3.3 aktuell ist und auch den SCSIDRV unterstützt. (Siehe HDDriver).

Mit dem CD Writer Suite Update Archive 3.2, welches man leicht und kostenlos für Besitzer der Version 3 von der Anodyne-Webseite herunterladbar ist, erhält man die Möglichkeit CD Writer Plus, das Programm mit dem sie bereits ISO9660-kompatible CDs geschrieben haben, auf die aktuelle Version 3.2 zu erweitern.

CD Writer Plus muß in der Version 2.1, mit der Orignaldiskette, zwingend vorhanden sein um das Update durchführen zu können. Bevor Sie damit beginnen irgendetwas von A nach B zu kopieren, sollten Sie READ_ME.1ST aus dem Updatearchiv gelesen haben. Es vereinfacht den Vorgang und spart unötige Fragen sowie Ärger.

Nach erfolgreichem Update, das Archiv enthält eine genaue Anleitung in englischer Sprache, zusätzlich zum neuen Handbuch, das man nur erhält wenn man das Paket auf dem Postweg geordert hat, entsteht aus ihrem CDWriter 2.1 zusätzlich das Programm CDISO in der aktuellen Version. CDISO erlaubt das vollstängige Erzeugen von ISO9660 CD-ROMs, auch mit langen Filenamen, 31-Zeichen, und mehr als 8 Ordnern in der Tiefe. Zusätzlich können und werden die Erweiterungen Joliet und Rock Ridge verwendet. Es ist unter den "Advanced Options" generell möglich auch Autorendaten auf die CD schreiben zu lassen. CD-R sowie CDRW können als Multisession XA geschrieben werden. Der einfache Datentransport geschieht mit Drag & Drop in das geöffnete CDISO Fenster, in einen dort eventuell vorhandenen Ordner, von jedem beliebigen GEM-Fenster aus.


- Technisches

Das erzeugen einer ISO9660 kompatiblen CD mit CDISO ist ein Arbeitsgang mit zwei Schritten. Ein ISO9660 Image-File wird auf der Festplatte angelegt, nach dem Abbild der Daten die sie in das geöffnete CDISO-Fenster kopiert haben. Ist dies geschehen kann dieses Image auf die eingelegte CD-R(W) kopiert werden.

Dazu ist das "Userinterface, bestehend aus CDISO.PRG/CDISO.RSC notwendig. MKISOFS.PRG wird von CDISO aufgerufen um ein ISO9660 Image zu erstellen. Alle diese Programm müssen daher im geleichen Verzeichniss stehen.


- Allgemeiner Betrieb

Im Betrieb legen sie einfach eine leere CD-R oder CDRW in ihren CD-Recorder. Schieben sie alle gewünschten Dateien auf das leere CDISO-Fenster, das offene leere Fenster ist die erste Session, je nachdem welche Einstellungen sie unter "General", "Drives" und "Advanced Options" vorgenommen haben, so wird hinterher ihre CD auch aussehen. Beachten Sie bitte das im CDISO-Fenster Ordner angelegt werden können die dann mit samt dem Inhalt, das was Sie hineinkopiert haben, so auf die CD geschrieben wird.

Haben Sie alle Daten arrangiert, nach ihrem ermessen, geben Sie dieser CD einen Namen und die CD kann beschrieben werden. Haben sie, später, den "Write- Protect"- Button, im Dialog "Advances Options" gesetzt, können nach der eben zuletzt geschriebenen Session keine weiteren Daten auf diese CD geschrieben werden. Die CD ist dann geschlossen.

Mit der ExtenDOS Version 3.3 haben diverse Features ihren Einzug gehalten. So wurde die Unterstützung für die CD-ROM Laufwerke Plextor PX4/12, PX8/20 und TEAC 50S, 55S ethabliert. Weiterhin wurde die SCSI-3/MMC kompatibilität erweitert so daß mehr SCSI-3/MMC kompatible Laufwerke direkt automatisch erkannt werden. Das trifft nun auch auf IDE-Geräte zu.

Da sich hier ständig Änderungen ergeben sollten sie hierzu immer zuerst die Read_Me Dateien bzw. das Handbuch zu ExtenDOS gelesen haben.
Ausschliesslich so erfahren sie aktuell ob und welches Laufwerk in welcher Form von ExtenDOS unterstützt wird.

Ab der Version 3.2B wird die SCSI Arbitration automatisch benutzt. Das ist immer dann der Fall wenn sich SCSI-2 Laufwerke automatisch auch als solche bei ExtenDOS identifizieren. Genauer bedeutet das für den Benutzer das er die Benutzung der "SCSI arbitration" unter dem Dialog "Erweiterte Einstellungen" im ExtenDOS-Installer nicht mehr extra einschalten muß. Dies geschieht nun automatisch.

Der eigentliche Treiber, CD.BOS, wurde um die Parameter "C=" und "M=" erweitert.

- C= Kontrolliert das SCSI Bus Parity Bit:
C=Yes bedeutet "Prüfen"
C=No bedeutet "Keine Prüfung"
Die Default-Einstellung ist "No"

- M= Kontrolliert den DMA-Transfer:
M=Yes bedeutet DMA-Transfer eingeschaltet, falls vorhanden.
M=Nn bedeutet Ausgeschaltet, kein DMA-Transfer
Die Default-Einstellung ist M=Yes


Bei DMA-Problemem sollten Sie "M=" auf keinen Fall benutzen!

UNIDRIVE.DOS hat zusätzliche Parameter erhalten:

- X= controls use of the RRIP extension information.
X=Y means use the RRIP data if available; X=N means don't use RRIP. The default is X=Y.

- U= controls the case of filenames returned by the
Fsfirst()/Fsnext() filesystem calls; it applies to filenames from standard ISO9660, Joliet, or RRIP. U=Y means force uppercase; U=N means leave as-is. The default is U=N.


Das Fastload Bit ist nun default gesetzt.(!nl)

Das "Rock Ridge Interchange Protocol" (RRIP), eine Alternative zum speichern langer Filenamen auf einer ISO9660 CD, ist in der Version 3.2B ethabliert. ExtenDOS Gold benutzt die RRIP Informationen (wenn Verfügbar) um lange Filenamen für MiNT/MagiC Filesysteme zu unterstützen. Das nächste ExtenDOS-Update wird die SCSIDRV-Unterstützung beinhalten.
